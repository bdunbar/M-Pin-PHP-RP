To sign a message (M), the Signer performs these steps:

  1. Choose a random (ephemeral) non-zero value
  2. Compute J = [j]G then assign to r the N-octet integer representing Jx;
  3. Compute a hash value HE = hash( HS || r || M )
  4. Verify that HE + r * SSK is non-zero modulo q else abort
  5. Compute modulo q; the Signer MUST then erase the value j
  6. If s' is too big to fit within an N-octet integer, then set the N-octet integer s = q - s'; otherwise, set the N-octet integers = s'
  7. Return the signature = ( r || s || PVT )

The SSK must be validated before being used as a signing key. Thus function uses the ID and the KPAK to validate a received (SSK,PVT) pair by performing these steps:

  1. Validate that the PVT lies on the elliptic curve E.
  2. Compute HS = hash( G || KPAK || ID || PVT ), an N-octet integer.
  3. Validate that KPAK = [SSK]G - [HS]PVT.

To verify a Signature ( r || s || PVT ) against a Signer's Identifier (ID), a message (M), and a pre-installed root of trust (KPAK), the Verifier must perform a procedure equivalent to the following:

  1. Check that the PVT lies on the elliptic curve E;
  2. Compute HS = hash( G || KPAK || ID || PVT );
  3. Compute HE = hash( HS || r || M );
  4. Y = [HS]PVT + KPAK;
  5. Compute J = [s]( [HE]G + [r]Y );
  6. Check that = r modulo p, and that modulo p is non-zero, before accepting the Signature as valid.
